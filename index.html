Trouble 01

재고실사 시 전 기능 동시 중지 처리

문제

재고실사 진행 중 다른 사용자가 입출고 등 기능을 계속 사용하면 재고 수치가 변경되어 실사 데이터의 정합성이 깨지는 문제 발생.

원인

실사 진행 상태를 단순 플래그로만 관리해 동시 접근을 완전히 차단하지 못하고, 이미 작업 중인 사용자에게 아무런 안내가 없었음.

해결

실사 시작 전 진행 중인 작업 여부를 체크하고, 사용 중인 사용자가 있을 경우 관리자에게 경고 팝업을 표시. 강제 시작 확인 시 실사 상태 플래그를 DB에 저장하고, 이후 모든 기능 요청에서 플래그를 조회하여 차단. 차단된 사용자에게는 "현재 재고실사가 진행 중입니다" 안내 메시지를 표시.

Trouble 02

SPC 관리도 이상 데이터 탐지 로직

문제

데이터가 누적될수록 SPC 관리도 차트 로딩이 느려지고, UCL/LCL 계산값이 표본 크기에 따라 다른 관리 계수(A2, D3, D4)를 반영하지 못해 오차가 발생.

원인

전체 데이터를 한 번에 로드하는 쿼리 구조와 관리 계수를 코드에 고정값으로 하드코딩한 것이 원인.

해결

최근 N개 데이터만 조회하는 쿼리로 변경해 로딩 성능을 개선하고, 표본 크기별 관리 계수를 별도 테이블로 분리하여 정확한 UCL/LCL 계산을 적용.

Trouble 03

파레트 할당 로직 복잡도

문제

여러 품목을 혼재 적재할 때 수량이 초과되거나 비효율적으로 분배되어 파레트 공간이 낭비되는 문제 발생.

원인

품목마다 적재 기준(규격, 무게, 단위)이 다른데도 동일한 기준을 일괄 적용하는 단순 로직으로 처리하고 있었음.

해결

품목별 적재 기준을 마스터 테이블로 분리하고, 혼재 적재 시 각 품목의 기준을 참조해 최적 배분을 계산하는 로직으로 개선.

💡

배운 점

제조 현장의 데이터는 실시간성과 정합성이 동시에 요구된다는 것을 체감했다. 단순 CRUD 구현에서 나아가, 동시성 제어·통계 계산·물류 최적화처럼 도메인 지식이 로직 설계에 직접 영향을 미치는 경험을 쌓을 수 있었다. 기능 개발 전에 업무 흐름을 먼저 파악하는 것이 얼마나 중요한지 배웠다.

Trouble 01

토큰 발행 후 특정 페이지 접근 차단 문제

문제

인증된 사용자임에도 특정 페이지 접근 시 차단되는 현상 발생. 로그인 자체는 정상 동작하지만 권한이 필요한 경로에서 접근이 거부됨.

원인

인증 토큰 검증은 정상적으로 수행되고 있었으나, 권한이 필요한 경로에 대한 접근 허용(permit) 설정이 라우팅 구성에서 누락되어 있었음.

해결

라우팅 및 권한 설정 구조를 전면 점검하고, 페이지별 접근 정책을 명확하게 재정의하여 적용. 단순 로그인 구현보다 권한 관리가 서비스 보안에 핵심적인 요소임을 인식.

Trouble 02

KCP 본인인증 JSP → Vue 전환 및 팝업 구조 개선

문제

KCP 본인인증은 JSP 기반 예제 코드를 제공하고 있었으나 프로젝트는 Vue.js 환경. 기존 예제를 그대로 적용할 수 없었고, 부모 페이지에서 인증 창을 직접 제어하는 것도 불가능했음.

해결

인증 요청 및 결과 처리 흐름을 Vue 컴포넌트 구조에 맞게 재구성. 별도의 외부 중간 페이지를 생성하여 해당 페이지에서 실제 KCP 인증 자식 창을 호출하는 방식으로 팝업 구조를 개선.

💡

배운 점

단순 기능 구현을 넘어, 외부 인증 서비스 연동 시 기술 스택 차이로 인한 문제를 분석하고 해결하는 과정을 경험했다. 토큰 개발 시 보안 관점에서 시스템을 설계하는 중요성을 배웠고, 레거시 기반 솔루션을 현재 환경에 맞게 재구성하는 과정에서 문제 해결 능력과 서비스 연동 역량을 높일 수 있었다.
